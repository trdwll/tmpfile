Frequently Asked Questions

How can I contact you?

XMPP/Jabber and Email - tmpfile@cock.li GPG/PGP (63D3 7F29 053B BEB6) and OTR FP (62EC78AE AA55C0B7 01A59131 93B93850 3A831B62)

We chose cock.li for our support email so we don't have to host our own or use some 3rd party that spys on you. Use GPG/PGP when mailing us (if possible) to ensure your communication with us is private.

How can I verify that you delete the files after 24 hours and don't store them elsewhere or send them to a 3 letter agency?

You can't, it's all about trust - you just have to take our word for it. We do recommend that you encrypt your files before uploading them so not even we could read the content. Trust but verify.

We run this cron find /path -type f -mmin +1440 -delete every hour to make sure the files get removed at ~24 hours.

Do you log anything?

We do not log any data. However, since you're uploading a file to our server - that's all that we store for 24 hours.

We open sourced the code to help ease your mind. We only keep nginx logs for 10 minutes then write them to /dev/null so we can do some debugging if an issue arises or if we need to ban a bad visitor.

Can I use a VPN/Tor/Proxy or Public Wifi while connected to a VPN and Tor while routing through a few other networks?

Yes, please do. - We do not block anything on our end.

Can I upload a file larger than 1GB?

Nope, sorry. (maybe in the future?)

Got an API?

No we do not. You can send a POST request to https://{{ MY_HOST_DOMAIN }} with the file param. It'll return a JSON Response.

curl -F "file=@mylocalfile.zip" https://{{ MY_HOST_DOMAIN }}

Why should I use {{ MY_HOST_DOMAIN }} over any other temporary file hosting services?

Use what you're comfortable with, we're not competing with any other services, but rather trying to strengthen the service idea as a whole. We think there should be more sites that store files temporarily.

What file types/content are allowed?

No. Not even the OS/partition. With OpenVZ that's essentially impossible. We'll probably move to KVM or a dedi in the future.
